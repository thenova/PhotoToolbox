# Photo Toolbox

A desktop photo management utility built with Electron, React, TypeScript, and Tailwind CSS.

## Tools

| Tool | Description |
|------|-------------|
| **Photo Sorter** | Copy photos from a source folder into one or more destinations, organized into `YYYY/MM/DD` subfolders using EXIF date (falls back to file modification time) |
| **EXIF Editor** | Read and edit EXIF tags on JPEG files (description, dates, artist, copyright, software) |
| **Photo Map** | Scan a folder tree recursively and display geotagged photos on an interactive map |
| **Photo Renamer** | Batch-rename photos using EXIF metadata patterns (date, camera make/model) |
| **Metadata Overview** | Aggregate EXIF statistics across a folder tree — cameras, lenses, apertures, shutter speeds, ISO, focal lengths, years, flash usage |

Supported formats: JPEG, PNG, GIF, BMP, WebP, TIFF, HEIC/HEIF, and RAW formats (CR2, CR3, NEF, ARW, DNG, RAF, ORF, RW2, and more).

## Prerequisites

- [Node.js](https://nodejs.org/) 18 or later
- npm (comes with Node.js)

## Getting started

```bash
# Install dependencies
npm install
```

## Development

```bash
npm run dev
```

Starts the Electron app in development mode with hot module replacement. The renderer runs on a local dev server; changes to React components and styles are reflected instantly without restarting.

## Building

```bash
npm run build
```

Compiles TypeScript and bundles all three targets (main, preload, renderer) into the `out/` directory using electron-vite.

## Creating an installer

```bash
npm run dist
```

Runs `build` then packages the app with electron-builder into `dist/Photo Toolbox-1.0.0-Setup.exe` (NSIS installer, x64).

### Windows symlink issue

On Windows, electron-builder downloads a `winCodeSign` toolset that contains macOS symlinks. Extracting these requires symlink privileges, which are off by default. If `npm run dist` fails with:

```
ERROR: Cannot create symbolic link : A required privilege is not held by the client.
```

**Fix (one-time):** Manually extract the archive excluding the darwin folder, then set `WIN_CODE_SIGN_PATH` to that directory:

```powershell
# Find the bundled 7-zip
$sevenZip = ".\node_modules\7zip-bin\win\x64\7za.exe"

# Pick any .7z from the winCodeSign cache
$archive = "$env:LOCALAPPDATA\electron-builder\Cache\winCodeSign\<hash>.7z"

# Extract, skipping darwin symlinks
$outDir = "$env:LOCALAPPDATA\electron-builder\Cache\winCodeSign\winCodeSign-2.6.0"
& $sevenZip x $archive "-o$outDir" -x'!darwin' -y

# Now build
$env:WIN_CODE_SIGN_PATH = $outDir
$env:CSC_IDENTITY_AUTO_DISCOVERY = "false"
npm run dist
```

Alternatively, enable **Windows Developer Mode** (Settings → Privacy & Security → For developers → Developer Mode) — this grants symlink privileges and lets the normal `npm run dist` work without any workaround.

## Project structure

```
photo-toolbox/
├── src/
│   ├── main/          # Electron main process (IPC handlers, file system, EXIF I/O)
│   ├── preload/       # Preload script (exposes IPC bridge to renderer)
│   ├── renderer/      # React UI
│   │   └── src/
│   │       ├── components/   # Sidebar navigation
│   │       ├── tools/        # One component per tool
│   │       └── App.tsx       # Root — tool switching
│   └── shared/        # Types shared between main and renderer
├── out/               # Compiled output (generated by build)
├── dist/              # Packaged installer (generated by dist)
└── electron.vite.config.ts
```

## Scripts reference

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `electron-vite dev` | Start in dev mode with HMR |
| `build` | `electron-vite build` | Compile to `out/` |
| `preview` | `electron-vite preview` | Preview the production build |
| `dist` | `build` + `electron-builder` | Build and package installer |
| `postinstall` | `electron-builder install-app-deps` | Rebuild native deps after `npm install` |

## Tech stack

- **Electron 33** — desktop shell
- **electron-vite 2** — build tooling and dev server
- **React 18** + **TypeScript 5** — renderer UI
- **Tailwind CSS 3** — styling
- **exifr** — EXIF reading
- **piexifjs** — EXIF writing (JPEG only)
- **Leaflet** / **react-leaflet** — map rendering
- **electron-builder 25** — packaging and NSIS installer
